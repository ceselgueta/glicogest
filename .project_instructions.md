# GlicoGest - Controle Glicêmico Gestacional

## Overview
App for gestational diabetes monitoring. Users register glucose readings (fasting + post-meal), view stats, generate PDF reports for their obstetrician.

## Tech Stack
- Next.js 14 with App Router
- Prisma ORM with PostgreSQL
- NextAuth.js v4 (credentials provider, JWT strategy)
- Tailwind CSS, Framer Motion, Lucide React icons
- react-hot-toast for notifications

## Design
- **Theme**: Pink/rose gradient (`from-pink-500 to-rose-500`)
- **Style**: White cards with soft shadows, rounded corners (2xl/3xl), backdrop blur headers
- **Language**: Portuguese (BR) throughout - all UI text in Portuguese
- **Font**: Inter

## Auth System
- Email + password authentication via NextAuth CredentialsProvider
- **Google SSO** via NextAuth GoogleProvider + PrismaAdapter
  - `allowDangerousEmailAccountLinking: true` (links Google to existing email accounts)
  - Account model in Prisma for OAuth persistence
  - JWT strategy with cookies config for state/pkce
  - GOOGLE_CLIENT_ID and GOOGLE_CLIENT_SECRET in .env (currently placeholders)
- Login page: `/login`, Signup page: `/signup` — both have "Entrar com Google" button
- Protected routes via middleware (matcher: `/dashboard/:path*`)
- Session includes: id, email, name, image, plan, planExpiresAt
- Auth config in `/lib/auth.ts`, types in `/types/next-auth.d.ts`
- Password is optional on User model (null for Google-only users)

## Plan System
- **free**: Patient registration + up to 7 days of records (distinct dates)
- **premium** (Plano Único Gestacional R$39.90): 6 months, unlimited records/PDFs/stats
- Plan enforcement in POST `/api/readings` and `/api/readings/batch`
- Payment integration (Stripe) planned for later

## Route Structure
- `/` - Public landing page (redirects to /dashboard if authenticated)
- `/login` - Login page
- `/signup` - Signup page (auto-login after signup)
- `/dashboard` - Main dashboard (protected) with readings, stats, patient summary

## API Routes (all require auth via getRequiredSession)
- `/api/readings` (GET/POST) - Glucose readings with userId filter + 7-day free limit
- `/api/readings/batch` (POST) - Batch readings with free plan limit
- `/api/readings/[id]` (DELETE) - Delete reading (ownership check)
- `/api/stats` (GET) - Statistics by reading type
- `/api/patient` (GET/POST) - Patient settings per user (claims legacy null-userId data)
- `/api/generate-pdf` (POST) - PDF report via Abacus HTML2PDF API
- `/api/signup` (POST) - User registration
- `/api/auth/[...nextauth]` - NextAuth handlers

## Database Models
- **User**: id, name, email, password, plan, planExpiresAt
- **GestationalGlucoseReading**: id, userId, readingDate, readingType (enum), valueMgDl, readingTime, notes
  - Unique constraint: [userId, readingDate, readingType]
- **PatientSettings**: id, userId, patientName, birthDate, pregnancyWeeks, estimatedDueDate, doctorName, fastingTarget, postMealTarget, postMealProtocol

## Alert System (lib/alerts.ts)
- Classifies glucose values into 7 alert levels:
  - 🟢 Normal (within target)
  - 🟡 Above target (> target and < 200)
  - 🟠 Hypoglycemia attention (< 70 and >= 54)
  - 🔴 Hypoglycemia serious (< 54 and >= 40)
  - 🚨 Hypoglycemia emergency (< 40)
  - 🟠 Very high glucose (>= 200 and < 250)
  - 🚨 High glucose emergency (>= 250 or severe symptoms)
- Alert modal shows after saving readings with non-normal values
- Symptoms tracking: 12 predefined symptoms (tremor, suor frio, fraqueza, etc.)
- Observations field for free-text notes (refeição, horário, etc.)
- Inline mini-alerts shown below input fields while typing
- Color-coded badges in readings table
- Medical disclaimer: "Este alerta é apenas informativo e não substitui avaliação médica."
- Alerts do NOT block saving measurements
- Alerts NOT shown in PDF reports

## Database Fields
- `symptoms` (String, nullable) - JSON array of symptom strings stored on GestationalGlucoseReading
- `observations` (String, nullable) - Free-text observations stored on GestationalGlucoseReading
- Extra data saved via PATCH `/api/readings/[id]/extra`

## Important Notes
- Legacy data (userId: null) is claimed by the first user who accesses patient settings
- Reading types: JEJUM, POS_CAFE_2H, POS_ALMOCO_2H, POS_JANTA_2H
- Protocol options: 1h or 2h (affects labels like "Pós-Café 1h" vs "Pós-Café 2h")
- Default targets: Fasting ≤95, Post-meal ≤120 mg/dL
